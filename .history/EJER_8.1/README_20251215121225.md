## **Ejercicio 1 – Carrito de la Compra Interactivo**

**Objetivo:** Crear un carrito que gestione productos, cantidades y precios de forma reactiva y optimizada.

**Hooks utilizados:**
- `useReducer`: Para manejar el estado complejo del carrito (añadir, eliminar, actualizar cantidad).
- `useCallback`: Para memoizar funciones que se pasan a componentes hijos (`addItem`, `increment`, `decrement`, `removeItem`) y evitar re-renders innecesarios.
- `useMemo`: Para calcular el precio total solo cuando cambian los items.

**Mejoras implementadas:**
- Evita re-renderizar todos los componentes al actualizar la cantidad de un solo producto.
- Mantiene la lógica del estado centralizada y limpia.

## **Ejercicio 2 – Dashboard de Tareas Multi-usuario**

**Objetivo:** Crear un panel de tareas donde el estado del usuario y de las tareas está desacoplado, con permisos según autor.

**Hooks y herramientas utilizadas:**
- `useContext`: Para separar `AuthContext` y `TasksContext`.
- `useReducer`: Para la lógica de las tareas (`ADD_TASK`, `TOGGLE_TASK`, `REMOVE_TASK`).
- `React.memo`: Para optimizar `TaskItem` y evitar re-render de tareas no afectadas al cambiar usuario.

**Mejoras implementadas:**
- Separación de contextos evita que un cambio de usuario provoque re-render de todas las tareas.
- `TaskItem` memoizado para solo renderizar tareas afectadas.

## **Ejercicio 3 – Editor de Temas UI**

**Objetivo:** Construir un editor de tema en tiempo real con múltiples componentes de vista previa que reaccionen solo a los cambios necesarios.

**Hooks y herramientas utilizadas:**
- `useReducer` + `ThemeContext`: Para manejar estado complejo del tema (`primaryColor`, `fontSize`, `textTransform`).
- `React.memo`: Para los componentes de vista previa (`BotonPreview`, `TextoPreview`, `CardPreview`) y evitar re-render innecesario.
- `useMemo`: Para calcular valores derivados (sombra de `CardPreview`) solo cuando cambia el color primario.

**Mejoras implementadas:**
- Solo los componentes que usan un valor del tema se re-renderizan al modificarlo.
- Evita recalcular sombras u otros valores derivados en cada render.
